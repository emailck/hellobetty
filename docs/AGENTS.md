## Purpose
- Store durable product scope, architecture, and API decisions.

## Ownership
- Own documents that explain current contracts across more than one implementation subtree.

## Local Contracts
- Describe implemented behavior as current and future behavior as planned.
- Remove stale decisions instead of preserving a change diary.

## Work Guidance
- Keep documents concise and link to source paths for implementation detail.

## Verification
- Re-read affected documents against the current code and commands.
